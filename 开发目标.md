## 一、产品定位

**一句话**：一本书进去 → **深度 Summary**（读前/读中材料）→ 阅读 + **Q&A**（对话）→ **Insight**（融会贯通）+ **Skill**（AI 工具包）。

### 三层概念（重要）

| 层 | 路径 | 何时产生 | 是什么 |
|----|------|----------|--------|
| **Preview** | `summary/overview.md` | 分章后 | 全书地图，决定值不值得深读 |
| **深度 Summary** | `summary/chapters/chNN.md` | Pipeline `deep` | 按章结构化笔记，**还不是 Insight** |
| **Insight** | `insight/qa.md` + `insight/synthesis.md` | **Q&A 之后** | 你的理解、融会贯通，个人知识库 |

> **深度 Summary ≠ Insight。**  
> Pipeline 只负责把书「读厚」成 Summary；Insight 必须来自你在 UI 里阅读时的真实提问与对话。

### Q&A 规则

- **不预生成问题** — pipeline / migrate 都不会写 Q&A
- 阅读时在右侧 **chatbot** 自然提问
- 每轮对话 **自动追加** 到 `insight/qa.md`，左侧导航「Q&A（阅读对话）」可查看
- `synthesis` **要求** `qa.md` 非空 — 没有你的 Q&A 就不该生成 Insight

### 两类交付物

| 交付物 | 路径 | 受众 |
|--------|------|------|
| **Summary + Insight** | `{book}NOTE/summary/` + `insight/` | 人 + PKM |
| **Skill** | `skill/` → `~/.cursor/skills/{slug}/` | AI Agent |

## 二、如何加入新书

### 方式 A · UI 导入（推荐）

1. `./run-ui.sh` → 打开 http://127.0.0.1:8765
2. 书架页点 **「+ 导入新书」**
3. 填书名、选源文件（**txt · md · docx · epub · pdf 文字层**）、勾选「自动分章」
4. **Input Pipeline** 自动标准化为纯文本 → 写入 NOTE 目录 → 可选分章
5. 导入成功后进入阅读器；或从书架卡片点开

未分章的书可在卡片上点 **「自动分章」**（识别「第 X 章/回/篇」标题）。

**扫描版 PDF**（无文字层，如杜秀古籍）→ 检测后提示需 **OCR 流程（P3）**，暂不支持直接导入。

### 方式 B · CLI

```bash
cd Readings/产品经理/book-compiler

# 1. 初始化 NOTE 目录（可选复制源 txt）
./run.sh init --title "精益创业" --slug lean-startup \
  --txt "../精益创业.txt" --type M

# 2. 分章 → 写入 _extract/ch*.txt（需按书写 split 脚本或手工切）
#    启示录已有：./run.sh split-inspired

# 3. Preview + Deep Summary
./run.sh preview --book lean-startup
./run.sh deep --book lean-startup --chapter ch01 --hitl
./run.sh deep --book lean-startup --all --batch

# 4. 阅读 UI：边读边问，Q&A 自动写入 insight/qa.md
./run-ui.sh

# 5. 有足够 Q&A 后，生成 Insight
./run.sh synthesis --book lean-startup
./run.sh compile --book lean-startup --install
```

---

`init` / UI 导入会创建：

```
{书名}NOTE/
├── _extract/              # 分章 OCR 原文（输入）
├── summary/               # Pipeline 产出：overview + chapters
├── insight/               # Q&A + synthesis（阅读之后）
│   └── book-meta.json
├── _state/
└── skill/
```

新书 slug 注册在 `book-compiler/_state/books.json`，UI 与 CLI 通过 `resolve_book_root(slug)` 发现。

**现有硬编码书**（无需 init）：`pm-book-sujie`、`inspired-cagan`。

## 三、目录结构（每本书）

```
{书名}NOTE/
├── _extract/ch*.txt          # OCR 分章原文
├── _state/pipeline.json
├── summary/                  # ← 深度 Summary（Q&A 之前）
│   ├── overview.md
│   ├── chapters/ch*.md
│   └── page-index.json       # 对话检索索引
├── insight/                  # ← Insight（Q&A 之后）
│   ├── book-meta.json
│   ├── qa.md                 # UI chatbot 自动追加，不预生成
│   ├── synthesis.md          # 融会贯通（需 qa.md）
│   ├── concept-index.json
│   └── concepts/*.md
└── skill/
```

**向后兼容**：旧书把 chapters 放在 `insight/chapters/` 仍能读；新书统一写 `summary/chapters/`。

## 四、Pipeline 流程

### 4.1 Input Pipeline（导入标准化）

```
源文件（txt / md / docx / epub / pdf）
  → ingest adapter（按格式解析）
  → BookDraft（canonical 纯文本 + 行锚点）
  → 写入 {书名}NOTE/{stem}.txt
  → split（分章 → _extract/ch*.txt）
```

| 格式 | Adapter | 状态 |
|------|---------|------|
| txt | 直接 UTF-8/GBK 解码 | ✅ |
| md | 去 frontmatter，保留正文 | ✅ |
| docx | zip + document.xml 抽段落 | ✅（stdlib） |
| epub | spine 顺序 HTML→文本 | ✅（stdlib） |
| pdf 文字层 | pypdf 抽字 + 页标记 | ✅（需 `pip install pypdf`） |
| pdf 扫描版 | 检测无文字层 | ⏳ P3 OCR 异步任务 |

代码：`src/book_compiler/ingest/`（`registry` + `adapters/*` + `pipeline.py`）

### 4.2 阅读 Pipeline

```
_extract/ 原文
  → preview（summary/overview.md）
  → deep × N 章（summary/chapters/chNN.md）
  → 【阅读 UI：chatbot Q&A → insight/qa.md】
  → synthesis（insight/synthesis.md，输入含 qa）
  → compile_skill → ~/.cursor/skills/{slug}/
```

### 4.3 阅读呈现

| 内容 | 存储 | 呈现 |
|------|------|------|
| Summary / Insight | `.md` | 浏览器 MD→HTML + `reader.css` |
| OCR 原文侧栏 | `.txt` + 行号 | 等宽纯文本，可点 L 锚点 |
| Chatbot 回复 | 对话记录 | MD→HTML 气泡 |

阅读器控件：**字号 A−/A+**、**纸书/夜间主题**（localStorage 持久化）。

## 七、设计原则

1. **Summary 厚、Insight 个人** — Pipeline 读厚；Insight 来自你的 Q&A
2. **Q&A 不预生成** — 问题在阅读中自然产生
3. **Grounded** — 答案有锚点/溯源
4. **不用 embedding** — PageIndex 推理检索
5. **简洁** — 侧栏一键、上原文下对话
