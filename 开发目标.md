## 一、产品定位

**一句话**：一本书进去 → **Insight**（给人读的个人知识库）+ **Skill**（给 AI 调用的工具包）。

不是通用阅读器，而是 **Book-to-Insight + Skill 编译器**，复刻并自动化你手工做《人人都是产品经理》NOTE 的工作流。

### 两类交付物

| 交付物 | 路径 | 受众 | 特点 |
|--------|------|------|------|
| **Insight** | `{book}NOTE/insight/` | 人 + PKM | 厚、有锚点、可跳转原文 |
| **Skill** | `skill/` → `~/.cursor/skills/{slug}/` | AI Agent | 薄、可触发、grounded 回答 |

### 两种阅读模式

| 模式 | 产出 | 时机 |
|------|------|------|
| **Preview** | `overview.md` | 分章后即可：决定值不值得深读 |
| **Deep** | `chapters/chNN.md` → `synthesis.md` | 逐章深读 + Q&A 闸门 + 全书融会贯通 |

### 模板 C（自动分型）

- **M 型**（方法论）：如《人人都是产品经理》— 概念清单 + 案例 + 表格
- **N 型**（叙事）：如《启示录》— 论点展开 + 书中原话 + 实践含义
- 按章 `template` 字段区分，pipeline 自动选择 prompt

## 二、目录结构（每本书）

```
{书名}NOTE/
├── _extract/ch*.txt          # OCR 分章原文（输入）
├── _state/pipeline.json      # 运行时状态（可重建）
├── insight/
│   ├── book-meta.json        # 元数据 + 章节索引 + pipeline 状态
│   ├── overview.md           # Preview：全书地图
│   ├── synthesis.md          # Deep 完成后：融会贯通
│   ├── qa.md                 # 全书 Q&A 日志
│   ├── page-index.json       # PageIndex 树索引（对话检索用）
│   ├── concept-index.json    # 概念 → 章节映射
│   ├── chapters/ch*.md       # 单章深度笔记
│   └── concepts/*.md         # 可选：单概念深页
└── skill/                    # compile 产出 → 安装到 ~/.cursor/skills/
```

## 四、核心架构

### 4.1 Pipeline 流程

```
源书 txt
  → split / migrate（分章）
  → preview（overview.md）
  → deep × N 章（chapters/chNN.md，HITL 可选）
  → synthesis（融会贯通）
  → compile_skill → ~/.cursor/skills/{slug}/
```

### 4.2 PageIndex 对话检索（无 embedding）

借鉴 [PageIndex](https://github.com/VectifyAI/PageIndex) 推理式检索，**不用向量**。

```
用户问题
  → 阶段1：LLM + 树骨架（标题+摘要）→ 推理选出 node_id 列表
  → 阶段2：按 node_id 拉取 insight 笔记片段
  → 阶段3：LLM 基于材料 + 对话历史 → 生成回答（Markdown）
```

树节点来源：

- `overview` / `synthesis`（全书级）
- `chNN`（章级）
- `chNN-sXX`（### 小节级）
- `concept-*`（概念页，如有）

索引文件：`insight/page-index.json`（首次对话时自动生成/缓存）。

### 4.3 overview vs synthesis

| | overview | synthesis |
|---|----------|-----------|
| **时机** | Preview（分章后） | Deep 全部完成后 |
| **输入** | `_extract/` 原文 | 各章 depth 笔记 + qa |
| **作用** | 进门地图：值不值得读 | 出门总报告：跨章融会贯通 |
| **对话检索** | 全书概览类问题 | 跨章主线/对比类问题 |

### 4.4 LLM 配置

```bash
# book-compiler/.env（gitignore，勿提交）
LLM_API_KEY=sk-...
LLM_BASE_URL=https://api.deepseek.com/v1
LLM_MODEL=deepseek-v4-flash
```

- **输出**：未设 `max_tokens`，用 API 默认值；只取 `content`，丢弃 `reasoning_content`
- **输入**：检索上下文约 18K 字符预算；对话历史最近 8 轮

## 九、设计原则

1. **Insight 厚、Skill 薄** — 人读厚的，AI 读薄的 + 按需查 Insight
2. **Grounded 优先** — 答案必须有锚点/溯源，不确定就说未涉及
3. **不用 embedding** — PageIndex 推理检索足够；结构化的书不需要向量
4. **HITL 不可省** — 深度笔记质量靠人审，batch 只是加速选项
5. **简洁优雅** — UI 不过度设计（侧栏一键、上原文下对话）；代码最小 diff
